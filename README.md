## 🚀 **Quick Start**

### **Prerequisites**
- Node.js 18+ (20+ recommended)
- npm/pnpm/yarn/bun
- MongoDB connection string
- GitHub/Google OAuth apps (optional)

### **🔧 Setup in 3 minutes**

```bash
# 1️⃣ Clone the repository
git clone https://github.com/AKRITI-SINGH1/Lilee.git
cd Lilee

# 2️⃣ Install dependencies
npm install
# or: pnpm install / yarn install / bun install

# 3️⃣ Environment setup
cp .env.example .env.local
# Edit .env.local with your values

# 4️⃣ Database setup
npx prisma generate
npx prisma db push

# 5️⃣ Launch development server
npm run dev
```

**🎉 Open [http://localhost:3000](http://localhost:3000) and start coding!**

## 🔐 **Environment Variables**

Create `.env.local` in the project root:

```bash
# 🗄️ Database
DATABASE_URL="mongodb+srv://username:password@cluster.mongodb.net/lilee?retryWrites=true&w=majority"

# 🔒 Authentication
AUTH_SECRET="your-super-secret-jwt-key-here"  # Generate: openssl rand -base64 32
NEXTAUTH_URL="http://localhost:3000"          # Production: https://yourdomain.com

# 🔗 OAuth Providers (Optional)
GITHUB_ID="your_github_oauth_client_id"
GITHUB_SECRET="your_github_oauth_client_secret"

GOOGLE_ID="your_google_oauth_client_id" 
GOOGLE_SECRET="your_google_oauth_client_secret"

# 🤖 AI Features (Optional)
OPENAI_API_KEY="sk-your-openai-api-key"      # For AI code suggestions
```

<details>
<summary><b>🔧 OAuth Setup Guide</b></summary>

### GitHub OAuth App
1. Go to [GitHub Developer Settings](https://github.com/settings/developers)
2. Create a new OAuth App
3. Set Homepage URL: `http://localhost:3000`
4. Set Authorization callback URL: `http://localhost:3000/api/auth/callback/github`

### Google OAuth App  
1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create a new project or select existing
3. Enable Google+ API
4. Create OAuth 2.0 credentials
5. Add `http://localhost:3000/api/auth/callback/google` to authorized redirect URIs

</details>

## 🧪 **Template System**

Lilee uses a **dual-template system** for maximum reliability:

### **🏠 Local Development**
- Full template files from `lileecode-starters/`
- Complete project structures with dependencies
- Fast iteration and customization

### **☁️ Production Deployment**  
- Embedded fallback templates in API routes
- Optimized for Vercel deployment limits
- Guaranteed availability and fast loading

### **🔄 How It Works**
1. **Template Request** → API checks for file-based templates
2. **Fallback Logic** → Uses embedded templates if files missing
3. **Smart Response** → Returns appropriate template with metadata
4. **User Feedback** → Clear notifications about template source

**Health Check:** Visit `/api/health/template` to verify template system status

<details>
<summary><b>🔧 Common Issues & Solutions</b></summary>

### **🖥️ Terminal Not Loading**
```bash
# Ensure WebContainers are supported
# Check browser compatibility (Chrome/Edge 86+, Firefox 89+)
# Verify container has non-zero height on mount
```

### **�️ Database Connection Issues**
```bash
# Regenerate Prisma client
npx prisma generate

# Sync database schema
npx prisma db push

# Reset database (⚠️ DANGER: deletes all data)
npx prisma migrate reset
```

### **🔐 OAuth Setup Problems**
- ✅ Check callback URLs match exactly
- ✅ Verify client ID/secret are correct
- ✅ Ensure OAuth apps are active
- ✅ Check NEXTAUTH_URL matches deployment URL

### **📦 Template Loading Failures**
- ✅ Templates should work via fallback system
- ✅ Check `/api/health/template` for system status  
- ✅ Verify authentication (templates require login)
- ✅ Clear browser cache and try again

### **🎨 UI Components Not Styling**
```bash

npm run build
```
